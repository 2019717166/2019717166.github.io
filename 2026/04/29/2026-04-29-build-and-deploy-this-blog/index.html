从零制作并部署这个 Hexo 博客

这篇文章记录这个博客从默认 Hexo 站点，逐步变成现在这个个人技术博客的过程。它不只是一次换主题，也包括站点定位、页面优化、本地写作管理和部署流程的整理。

目标

一开始的目标很明确：把默认的 Hexo 博客优化成一个适合长期维护的技术博客。

我希望它具备这些能力：

  • 适合写技术文章，阅读体验清爽。
  • 支持分类、标签、归档和本地搜索。
  • 有个人主页信息，能展示博客方向和联系方式。
  • 首页不只是文章列表，也能体现人工智能、Web 和工程实践方向。
  • 能在本地通过一个简单界面写文章、编辑配置，并自动生成预览。
  • 最终部署到 GitHub Pages，访问成本低，维护简单。

切换到 Fluid 主题

默认的 Landscape 主题比较基础，所以我把主题切换为 Fluid。Fluid 更适合中文技术博客，支持暗色模式、本地搜索、代码块增强、文章目录、关于页等功能。

这次搭建的收获

这次搭建不只是把主题换好，更重要的是把后续维护流程也整理清楚了。

现在的博客已经具备了一个长期技术记录站需要的基础能力：

  • 写作入口明确。
  • 页面结构清晰。
  • 搜索不会被样式和脚本污染。
  • 首页能体现技术方向。
  • 本地管理界面能降低写文章和改配置的成本。
  • 部署流程保持简单，仍然适合 GitHub Pages。

后续只需要持续补充文章内容，把学习人工智能、Web 开发和工程实践的过程记录下来，这个博客就会慢慢变成一份真正有价值的个人知识库。
